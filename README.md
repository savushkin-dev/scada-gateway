# scada-gateway
Модуль обмена данными с контроллером

Минимальный план
•  (+) Поддержка протокола OPC UA обмена данными с контроллером;
•  (+) Проверка работы на симуляторе контроллера;
•  Чтение описания из базы данных и построение на сервере объектной модели базы каналов;
•  Передача значений тегов между серверами работы с контроллерами и серверами работы с клиентами через шину данных на Apache Kafka;
•  Обработка и форматирование данных для передачи в Kafka;
•  Ведение журнала событий;

Оптимистичный план
•  Подключение пользовательских скриптов для обработки значений тегов;
•  Поддержка протоколов Mudbus, Profinet обмена данными с контроллером;
•  Анализ журнала событий с помощью искусственного интеллекта.
